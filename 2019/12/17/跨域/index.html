跨域访问

发表于 2019-12-17 分类于 JS

同源策略是浏览器出于安全考虑的一个基本策略，它会限制来自不同的源的脚本对当前页面的修改和读取，简单来说就是协议，主域名，子域名，端口号，任意一个条件不同，就是不同的源，无法读取和修改另一个页面的内容

但在浏览器中 script img link iframe 四种标签可以不受同源策略的限制，加载资源。

跨域的方法

JSONP原理：利用了script标签，使网页可以动态获到其他页面的json数据。基本使用方法有

1.利用ajax把数据类型改为jsonp

2.发送带有回调函数作为参数，服务端数据拼凑到回调函数中，返回给浏览器，浏览器解析执行，前端就拿到回调函数返回的数据。

缺点是只能使用Get方法

CORS原理：
基于JS无法控制HTTP头部，XMLHttpRequest对象是可以进行跨域访问，但是需要通过目标域返回的HTTP头部来授权，一般由后端来设置HTTP头部

WebSocket原理：是HTML5一种协议。它实现了浏览器与服务器全双工通信，同时允许跨域通讯。WebSocket和HTTP都是基于TCP协议，在建立连接以后服务端和客户端都能主动发送或者接收数据

postMessage方法：基于iframe标签，原本iframe窗口和windows窗口之间是无法通信的，HTML5则引入了一个全新的API即window.postMessage。

postMessage方法的第一个参数是具体的信息内容，第二个参数是接收消息的窗口的源（origin），即”协议 + 域名 + 端口”。也可以设为*，不限制域名。
